JS Interpreter - Notes & Limitations

NOTE: Before using the JavaScript Interpreter, keep the following limitations in mind:

  • If you use infinite loops or conditional loops, all HID instructions must use await. Otherwise, the UI will freeze.
  • Parallel execution is not supported. Using Promise.all() or concurrent calls may cause mixed commands or ACK errors.
  • Async functions are allowed, but they must always be awaited. They do not behave like standard JavaScript async functions and may mix HID commands if misused.
  • Access to browser APIs is blocked. You cannot use document, window, fetch, XHR, WebRTC, etc.
  • Infinite loops and await are not allowed during conversion to DuckyScript.
  • The function repeat(count, fn) is only available for the DuckyScript converter. It does not execute commands during JS runtime execution.
  • repeat() does not generate the native REPEAT command. Instead, it expands the commands inline the specified number of times.
  • Some DuckyScript commands are not supported in this execution mode: LSTRING, REPEAT, LOOP, DELAY.
  • This execution mode is slower than others, but allows more flexibility and the use of JavaScript logic.
  • Please also note that this mode can be somewhat unstable and may occasionally produce a TIMEOUT error.
  • Improvements are planned for the future.

To enable disabled keyboards, uncomment the line in locale/locale.cpp where they appear. Be mindful of space limitations; ideally, only comment out keyboards you don't use frequently. You can set approximately 3 or 4 keyboards.
